§5 · D3.4 — Interoperabilità IoT (RI)

Framework IoT: OPC-UA · Pro-face Connect · Middleware · KPI · OEE

Il deliverable D3.4 definisce l'architettura completa del framework di integrazione delle IoT Resource dell'impianto produttivo. Focus su: raccolta dati, preprocessing, schedulazione ordini, monitoraggio avanzamento, KPI, OEE, manutenzione predittiva.

Architettura connettore IoT · Pro-face Connect
[PLC / CNC / Sensori]
   ↓ Ethernet industriale
[HMI Pro-face + GP-Pro EX]
   ↓ Pro-face Connect Agent · Porta 11444
[Cloud Gateway Pro-face Connect]
   ↓ TLS VPN · opc.tcp://127.0.0.1:4840
[OPC-UA Client → Analytics Python]
Esempio connessione Python
# OPC-UA Client Python
from opcua import Client
client = Client("opc.tcp://127.0.0.1:4840")
client.connect()
volt = client.get_node("ns=2;s=FBK_VOLT").get_value()
# → 12.32 V

§7 · Proof of Concept — Demo Data Analytics

Dimostratore Python: Web App · ML · Dashboard KPI saldatura laser

L'applicazione dimostrativa è una Web App completa (Python + Streamlit) che integra analisi esplorativa, visualizzazione dati e machine learning su dataset simulato compatibile con la saldatura laser TR4.0. Include scenario "aumentato" con sistema di acquisizione immagini per rilevamento difetti.

Dataset utilizzato: 1.000 osservazioni simulate realistiche, coprendo l'intero processo (impostazioni macchina laser, condizioni ambientali, risultati saldatura). Target ML: predizione larghezza cordone (LC) e penetrazione (PE). Il dataset è compatibile con i parametri reali della cella TR4.0.

-- TECNOMULIPAST · PostgreSQL Schema v2 · Digital Factory Data Model

-- ─────────────────────────────────────────────────────────────────
-- LIVELLO OT: Registro fisico degli impianti
-- Rappresenta la cella robotica come entità di primo livello
-- ─────────────────────────────────────────────────────────────────

CREATE TABLE celle_robotiche (
  id                  SERIAL PRIMARY KEY,
  nome_cella          TEXT NOT NULL,             -- es. 'TECNOROBOT TR4.0'
  modello             TEXT NOT NULL,             -- es. 'TR4.0'
  produttore          TEXT DEFAULT 'Tecnorobot',
  robot               TEXT NOT NULL,             -- es. 'Fanuc 6-axis'
  plc                 TEXT NOT NULL,             -- es. 'Fanuc PLC'
  hmi                 TEXT,                      -- es. 'Pro-face GP-Pro EX'
  sorgente_laser      TEXT,                      -- es. 'IPG Photonics YLS'
  opcua_endpoint      TEXT,                      -- es. 'opc.tcp://127.0.0.1:4840'
  opcua_namespace     INTEGER DEFAULT 2,         -- NS2 per TECNOROBOT
  gateway             TEXT DEFAULT 'Pro-face Connect',
  porta_gateway       INTEGER DEFAULT 11444,
  porta_https         INTEGER DEFAULT 443,
  ubicazione          TEXT,                      -- es. 'Reparto saldatura · Gravina'
  stato_operativo     TEXT CHECK (stato_operativo IN (
                        'attiva','manutenzione','ferma','in_collaudo'
                      )) DEFAULT 'attiva',
  data_installazione  DATE,
  matricola           TEXT UNIQUE,               -- usata nel nome DB: ROBOT_E+matricola
  db_robot_name       TEXT,                      -- es. 'ROBOT_E12345'
  db_login            TEXT DEFAULT 'customer',
  note                TEXT,
  created_at          TIMESTAMPTZ DEFAULT NOW(),
  updated_at          TIMESTAMPTZ DEFAULT NOW()
);

-- ─────────────────────────────────────────────────────────────────
-- LIVELLO OT: Catalogo tag OPC-UA e sensori fisici
-- Ogni sensore/tag è associato alla cella specifica
-- ─────────────────────────────────────────────────────────────────

CREATE TABLE sensori_iot (
  id                       SERIAL PRIMARY KEY,
  cella_id                 INTEGER NOT NULL REFERENCES celle_robotiche(id) ON DELETE CASCADE,
  node_id_opcua            TEXT NOT NULL,        -- es. 'ns=2;s=SQL_FBK_VOLT'
  nome_tag                 TEXT NOT NULL,        -- es. 'SQL_FBK_VOLT'
  prefisso                 TEXT CHECK (prefisso IN ('INT_','SQL_')),
  -- INT_* = scrittura JERP→Robot; SQL_* = lettura Robot→JERP
  descrizione              TEXT,
  tipo_dato                TEXT CHECK (tipo_dato IN (
                             'tensione','corrente','velocita_filo',
                             'tempo_arco','stato_macchina','allarme',
                             'quantita','flag_avanzamento','temperatura',
                             'potenza_laser','impulso','frequenza',
                             'diametro_fascio','larghezza_cordone',
                             'penetrazione','generico'
                           )) NOT NULL,
  unita_misura             TEXT,                 -- V, A, s, mm, µm, W, Hz, ms, —
  data_type_opcua          TEXT CHECK (data_type_opcua IN (
                             'Boolean','Int16','Int32','Float',
                             'Double','String','DateTime'
                           )),
  frequenza_campionamento_ms INTEGER DEFAULT 1000,-- intervallo polling ms
  soglia_alert_min         REAL,
  soglia_alert_max         REAL,
  attivo                   BOOLEAN DEFAULT TRUE,
  note                     TEXT,
  UNIQUE (cella_id, nome_tag)
);

-- ─────────────────────────────────────────────────────────────────
-- LIVELLO IT/ERP: Ordini di produzione
-- Compilata dal JERP; inviata al robot via INT_* su OPC-UA
-- ─────────────────────────────────────────────────────────────────

CREATE TABLE odp (
  id                   SERIAL PRIMARY KEY,
  cella_id             INTEGER REFERENCES celle_robotiche(id),
  programma            TEXT NOT NULL,            -- INT_ODPx_PROG
  commessa             TEXT NOT NULL,            -- INT_ODPx_COMM
  des_prod             TEXT,                     -- INT_ODPx_DESCR
  articolo             TEXT,                     -- INT_ODPx_ART
  quant_da_produrre    INTEGER NOT NULL,         -- INT_ODPx_QUANT
  id_odp_robot         INTEGER,                  -- INT_ODPx_ID (slot 1-8 sul robot)
  slot_odp             INTEGER CHECK (slot_odp BETWEEN 1 AND 8),
  stato                TEXT CHECK (stato IN (
                         'in_attesa','sincronizzato','in_lavorazione',
                         'completato','annullato'
                       )) DEFAULT 'in_attesa',
  sync_robot           BOOLEAN DEFAULT FALSE,
  sync_timestamp       TIMESTAMPTZ,
  jerp_commessa_ref    TEXT,                     -- chiave JERP per riconciliazione
  created_at           TIMESTAMPTZ DEFAULT NOW()
);

-- ─────────────────────────────────────────────────────────────────
-- LIVELLO OT/IT: Avanzamenti lavorazioni
-- Scritti dal robot via SQL_* su OPC-UA → SQL Server → PostgreSQL
-- Ogni record = evento robot (produzione o diagnostica)
-- ─────────────────────────────────────────────────────────────────

CREATE TABLE avanzamento (
  id                   BIGSERIAL PRIMARY KEY,
  odp_id               INTEGER REFERENCES odp(id),
  cella_id             INTEGER NOT NULL REFERENCES celle_robotiche(id),
  -- Timestamp (ricostruito dai campi giorno/ora del robot)
  data_gg              SMALLINT, data_mm SMALLINT, data_aa SMALLINT,
  hh                   SMALLINT, min SMALLINT, sec SMALLINT,
  timestamp_calc       TIMESTAMPTZ GENERATED ALWAYS AS (
                         MAKE_TIMESTAMP(
                           data_aa + 2000, data_mm, data_gg, hh, min, sec
                         )
                       ) STORED,
  -- Contesto operativo
  operatore            TEXT,                     -- SQL_OPERATORE
  stazione             SMALLINT,                 -- SQL_STAZIONE (1 o 2)
  programma            TEXT,
  commessa             TEXT,
  articolo             TEXT,
  seq_prog_num         SMALLINT,
  seq_prog_descr       TEXT,
  -- Produzione
  tempo_esecuzione     REAL,                     -- SQL_TEMPO_ESEC (secondi)
  qta_prodotta         SMALLINT,                 -- SQL_QTA_PRODOTTA
  flag_finito          SMALLINT,                 -- SQL_FLAG_FINITO: 1=avvio, 199=fine Staz1, 299=fine Staz2
  id_sql               SMALLINT,                 -- FK logica verso odp.id_odp_robot
  -- Parametri saldatura (MIG/MAG)
  fbk_volt             REAL,                     -- SQL_FBK_VOLT (V) · media: 12,32V
  fbk_amp              REAL,                     -- SQL_FBK_AMP (A) · media: 78,56A
  cmd_wfs              REAL,                     -- SQL_CMD_WFS (velocità filo m/min)
  arc_on_h             SMALLINT,                 -- SQL_ARCON_H
  arc_on_m             SMALLINT,                 -- SQL_ARCON_M
  arc_on_s             SMALLINT,                 -- SQL_ARCON_S
  arc_on_sec_tot       INTEGER GENERATED ALWAYS AS (
                         COALESCE(arc_on_h,0)*3600 +
                         COALESCE(arc_on_m,0)*60   +
                         COALESCE(arc_on_s,0)
                       ) STORED,
  -- Parametri saldatura laser (dataset simulato · 1.000 record)
  pl_potenza           REAL,                     -- Potenza laser (W)
  di_durata            REAL,                     -- Durata impulso (ms)
  fi_frequenza         REAL,                     -- Frequenza impulsi (Hz)
  df_diametro_fascio   REAL,                     -- Diametro fascio (µm)
  ve_velocita          REAL,                     -- Velocità avanzamento (mm/s)
  an_angolo            REAL,                     -- Angolo incidenza (°)
  te_temperatura       REAL,                     -- Temperatura ambiente (°C)
  -- Output qualità (target ML)
  lc_larghezza         REAL,                     -- Larghezza cordone (µm) ← TARGET RF/GB/SVM
  pe_penetrazione      REAL,                     -- Penetrazione (mm)
  -- Allarmi (diagnostica · is_diagnostica=TRUE se allarme1 IS NOT NULL)
  allarme1             TEXT, allarme2  TEXT, allarme3  TEXT,
  allarme4             TEXT, allarme5  TEXT, allarme6  TEXT,
  allarme7             TEXT, allarme8  TEXT, allarme9  TEXT,
  allarme10            TEXT,
  is_diagnostica       BOOLEAN GENERATED ALWAYS AS (allarme1 IS NOT NULL) STORED,
  -- Costi (dal dataset simulato)
  costo_eur            NUMERIC(10,2),            -- media €132,38
  prezzo_vendita_eur   NUMERIC(10,2)             -- media €206,08
);

-- ─────────────────────────────────────────────────────────────────
-- LIVELLO IoT: Log time-series OPC-UA arricchito
-- Un record per ogni lettura di tag OPC-UA
-- Sostituisce il log_iot precedente (troppo generico)
-- ─────────────────────────────────────────────────────────────────

CREATE TABLE log_iot (
  id                   BIGSERIAL PRIMARY KEY,
  cella_id             INTEGER NOT NULL REFERENCES celle_robotiche(id),
  sensore_id           INTEGER REFERENCES sensori_iot(id),
  node_id              TEXT NOT NULL,            -- ns=2;s=SQL_FBK_VOLT
  nome_tag             TEXT,                     -- SQL_FBK_VOLT
  valore_raw           TEXT NOT NULL,            -- valore grezzo come stringa
  valore_num           DOUBLE PRECISION,         -- cast numerico quando applicabile
  unita_misura         TEXT,                     -- V, A, mm/min, s, W, Hz, —
  tipo_dato            TEXT CHECK (tipo_dato IN (
                         'tensione','corrente','velocita_filo',
                         'tempo_arco','stato_macchina','allarme',
                         'quantita','flag_avanzamento','temperatura',
                         'potenza_laser','impulso','frequenza',
                         'diametro_fascio','larghezza_cordone',
                         'penetrazione','generico'
                       )),
  qualita_dato         TEXT CHECK (qualita_dato IN (
                         'good','uncertain','bad','timeout','simulated'
                       )) DEFAULT 'good',
  sorgente             TEXT CHECK (sorgente IN (
                         'opcua_polling','opcua_subscription',
                         'mqtt','rest_api','simulato','manuale'
                       )) DEFAULT 'opcua_polling',
  timestamp_robot      TIMESTAMPTZ,              -- timestamp originale dal robot
  timestamp_server     TIMESTAMPTZ DEFAULT NOW(),-- timestamp ricezione server
  latenza_ms           INTEGER GENERATED ALWAYS AS (
                         EXTRACT(MILLISECONDS FROM
                           (timestamp_server - timestamp_robot))::INTEGER
                       ) STORED,
  odp_ref              TEXT,                     -- commessa/programma correlato
  avanzamento_id       BIGINT REFERENCES avanzamento(id)
);

-- ─────────────────────────────────────────────────────────────────
-- LIVELLO ANALYTICS: KPI calcolati per periodo / cella
-- Alimentato da job periodici su avanzamento + log_iot
-- ─────────────────────────────────────────────────────────────────

CREATE TABLE kpi_qualita (
  id                   BIGSERIAL PRIMARY KEY,
  cella_id             INTEGER NOT NULL REFERENCES celle_robotiche(id),
  periodo_da           TIMESTAMPTZ NOT NULL,
  periodo_a            TIMESTAMPTZ NOT NULL,
  granularita          TEXT CHECK (granularita IN ('turno','giorno','settimana','mese')),
  -- KPI saldatura (MIG/MAG + Laser)
  volt_medio           REAL, volt_sigma         REAL,
  amp_medio            REAL, amp_sigma          REAL,
  wfs_medio            REAL,
  arc_on_sec_totale    INTEGER,
  arc_sat_pct          REAL,                    -- saturazione arco % = arc_on/tempo_tot
  -- KPI produzione
  n_cicli              INTEGER,
  n_cicli_produttivi   INTEGER,
  n_cicli_diagnostici  INTEGER,
  throughput_pz_ora    REAL,
  tempo_ciclo_medio    REAL,
  tempo_ciclo_sigma    REAL,
  -- KPI qualità
  tasso_nc_pct         REAL,                    -- NC%: allarme≠NULL / n_cicli
  n_allarmi            INTEGER,
  lc_medio             REAL,                    -- larghezza cordone µm
  lc_sigma             REAL,
  pe_medio             REAL,                    -- penetrazione mm
  -- OEE (Overall Equipment Effectiveness)
  oee_disponibilita    REAL,                    -- uptime / tempo_schedulato
  oee_prestazione      REAL,                    -- cicli_reali / cicli_teorici
  oee_qualita          REAL,                    -- pz_conformi / pz_prodotti
  oee_totale           REAL GENERATED ALWAYS AS (
                         ROUND((COALESCE(oee_disponibilita,0) *
                                COALESCE(oee_prestazione,0) *
                                COALESCE(oee_qualita,0))::NUMERIC, 4)
                       ) STORED,
  -- Machine Learning
  r2_modello           REAL,                    -- R² ultimo modello ML
  algoritmo_ml         TEXT,                    -- RF | GB | SVM | Ridge | Lasso | Linear
  rmse_modello         REAL,
  features_usate       TEXT[],                  -- array colonne usate nel training
  -- Manutenzione predittiva
  mtbf_h               REAL,                    -- Mean Time Between Failures (ore)
  mttr_min             REAL,                    -- Mean Time To Repair (minuti)
  score_salute_cella   REAL CHECK (score_salute_cella BETWEEN 0 AND 100),
  created_at           TIMESTAMPTZ DEFAULT NOW()
);

-- ─────────────────────────────────────────────────────────────────
-- LIVELLO ANALYTICS: Modelli ML addestrati
-- Traccia versioni, metriche e hyperparameter di ogni run
-- ─────────────────────────────────────────────────────────────────

CREATE TABLE modelli_ml (
  id                   SERIAL PRIMARY KEY,
  cella_id             INTEGER REFERENCES celle_robotiche(id),
  algoritmo            TEXT NOT NULL,            -- RandomForest | GradientBoosting | SVM | Ridge | Lasso | Linear
  versione             TEXT NOT NULL,
  target               TEXT NOT NULL,            -- lc_larghezza | pe_penetrazione | tasso_nc
  features             TEXT[] NOT NULL,
  n_campioni_train     INTEGER,
  n_campioni_test      INTEGER,
  r2_train             REAL, r2_test            REAL,
  rmse_train           REAL, rmse_test          REAL,
  mae_test             REAL,
  hyperparameters      JSONB,                    -- {"n_estimators":100,"max_depth":5,...}
  feature_importance   JSONB,                    -- {"pl_potenza":0.32,"di_durata":0.21,...}
  dataset_hash         TEXT,                     -- SHA256 del dataset usato
  addestrato_da        TEXT,
  addestrato_il        TIMESTAMPTZ DEFAULT NOW(),
  in_produzione        BOOLEAN DEFAULT FALSE,
  note                 TEXT
);

Criticità dello schema v1
Assenza entità fisica: nessuna tabella rappresentava la cella robotica. Il modello iniziava da odp, saltando il livello OT.
log_iot troppo generico: solo node_id, valore, timestamp. Impossibile distinguere tensione da allarme da flag di avanzamento. Nessuna qualità del dato, nessuna sorgente, nessuna latenza.
Nessun catalogo OPC-UA: i 92 nodi variabile del TECNOROBOT non erano mappati. Nessuna traccia di INT_* vs SQL_*, frequenze di campionamento, soglie di alert.
KPI non calcolabile automaticamente: OEE non aveva formula in DB. R² e algoritmo ML non tracciati per versione. Nessun MTBF, nessun score salute.
Nessun tracciamento ML: i modelli Random Forest, Gradient Boosting, SVM non avevano persistenza delle run, hyperparameter o feature importance.
Soluzioni introdotte in v2
celle_robotiche come entità radice: tutta la gerarchia dipende da essa. cella → sensori_iot, odp, avanzamento, log_iot, kpi_qualita formano un grafo coeso. Multi-cella pronto (es. aggiunta OR2 piegatura).
sensori_iot come catalogo OPC-UA: ogni tag ha node_id, tipo (16 enum), unità, frequenza, soglie. Permette alert automatici e dashboard parametrica. Seed data include tutti i 92 nodi principali.
log_iot arricchito: qualita_dato (OPC-UA Quality Status), sorgente (polling vs subscription vs MQTT vs simulato), latenza calcolata come GENERATED COLUMN. Consente filtrare dati di bassa qualità nei KPI.
OEE come GENERATED COLUMN: oee_totale = disponibilita × prestazione × qualita calcolato automaticamente dal DB. Granularità turno/giorno/settimana/mese per analisi temporale.
modelli_ml con versioning: ogni run ML ha algoritmo, versione, features[], r2/rmse, hyperparameters JSONB, feature_importance JSONB, dataset_hash. Permette audit scientifico e confronto modelli nel tempo.
